## 🎨 Design system centralizado

**Objetivo:** permitir customização visual completa do app editando apenas um arquivo CSS, sem mexer em componentes.

### Arquivos
- `src/styles/theme.css` — **único arquivo a editar pra mudar layout**. Contém todas as cores, espaçamentos, raios, sombras, tipografia. Cada seção comentada explicando o que afeta.
- `src/index.css` — agora importa theme.css + define classes de componentes (btn, card, chip, input) que consomem as vars do tema.
- `tailwind.config.js` — colors mapeadas pra CSS vars (Tailwind classes como `bg-brand-600` continuam funcionando, mas pegam valores do theme.css).

### Como funciona
1. `theme.css` define `--color-brand-600: #1873f5` em `:root` (claro) e em `.dark` (escuro).
2. `tailwind.config.js` mapeia `brand-600` pra `var(--color-brand-600)`.
3. Componentes usam classes Tailwind (`bg-brand-600`) ou diretas (`background: var(--color-brand-600)`).
4. **Editou theme.css → tudo no app muda automaticamente.**

### Variáveis disponíveis (resumo)

| Categoria | Vars |
|---|---|
| Cores brand | `--color-brand-50` até `--color-brand-900` |
| Cores semânticas | `--color-success`, `--color-warning`, `--color-danger`, `--color-info` |
| Cores neutras | `--color-bg`, `--color-bg-elevated`, `--color-text`, `--color-text-muted`, `--color-border` |
| Tipografia | `--font-family`, `--font-size-base`, `--font-weight-bold` |
| Espaçamento | `--space-xs` até `--space-2xl` |
| Raios | `--radius-sm`, `--radius-md`, `--radius-lg`, `--radius-xl`, `--radius-pill` |
| Sombras | `--shadow-sm`, `--shadow-md`, `--shadow-lg`, `--shadow-elevated` |
| Animações | `--transition-fast`, `--transition-base`, `--transition-slow` |

Ver `src/styles/theme.css` para lista completa + comentários explicativos.

## 🎨 Sistema de ícones flat (lucide)

### Componente Icon central
`src/components/Icon.jsx` mapeia nomes semânticos → ícones SVG (`lucide-react`) com fallback pra emoji legado.

**Uso:**
```jsx
<Icon name="pill" size={20} />
<Icon name="alarm" className="text-rose-500" strokeWidth={2.5} />
```

### Toggle reversível
- **Default:** flat (lucide SVGs)
- **Reverter pra emojis legados:** Settings → Aparência → Estilo de ícones → Emoji
- **Via DevTools:** `localStorage.setItem('dosy_icon_style', 'emoji')` + reload

## 📢 Publicidade em todas as telas (in-flow, topo)

- `<AdBanner />` montado por página, **logo abaixo do header** (e do FilterBar no Dashboard), centralizado dentro do container `max-w-md`.
- **Native (Android):** AdMob banner overlay TOP_CENTER, margin 56dp abaixo do AppHeader. Singleton via `window.__dosyAdMobShown` flag → `AdMob.initialize` + `showBanner` chamados uma vez por sessão; `removeBanner` NÃO chamado no unmount (evita flicker entre páginas). Componente in-flow no native = `null` (overlay já é global).
- **Web:** card AdSense ou placeholder dentro do flow normal da página.
- Free users only (`useShowAds`). PRO/Plus/Admin → null.
- Páginas com mount: Dashboard (após Stat row), More, Patients, PatientDetail, PatientForm, TreatmentForm, TreatmentList, SOS, DoseHistory, Reports, Analytics, Settings.
- Páginas sem mount: Admin (admin não vê ads), Login/Install/ResetPassword/Termos/Privacidade (auth/legal flows).

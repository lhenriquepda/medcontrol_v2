## Motivação

Pós-refactor #209 (v0.2.1.9), arquitetura de alarmes ficou com 4 paths coordenados (JS rescheduleAll + dose-trigger-handler + daily-alarm-sync + DoseSyncWorker) + AlarmReceiver fire + FCM cancel handler. Sem visibilidade end-to-end:
- Algum path agendando duplicado?
- Cancel chegou ao device?
- Cron 5am realmente disparou FCM data?
- Worker rodou nas últimas 24h?
- Alarme fired_received aconteceu?

User pediu sistema observabilidade em `admin.dosymed.app`.

## Implementação

### 1. DB schema (migration `create_alarm_audit_log_v0_2_2_0`)

```sql
medcontrol.alarm_audit_log (
  id uuid PK, user_id uuid FK, device_id text, dose_id uuid,
  source text CHECK IN ('js_scheduler','java_alarm_scheduler','java_worker',
                        'java_fcm_received','edge_daily_sync','edge_trigger_handler'),
  action text CHECK IN ('scheduled','cancelled','fired_received','fcm_sent',
                        'skipped','batch_start','batch_end'),
  scheduled_at timestamptz, patient_name text, med_name text,
  metadata jsonb DEFAULT '{}', created_at timestamptz DEFAULT now()
)

medcontrol.alarm_audit_config (
  user_id uuid PK FK, enabled boolean DEFAULT true,
  notes text, created_at, created_by, updated_at
)
```

**RLS:** admin-only SELECT logs + manage config. User INSERT próprio gated por `EXISTS (SELECT 1 FROM alarm_audit_config WHERE user_id = auth.uid() AND enabled = true)`.

**Helper RPC:** `is_alarm_audit_enabled(p_user_id)` consulted pelo app/Edge (cache 5min JS-side).

**Admin RPCs:**
- `admin_list_alarm_audit(filtros)` — lista paginada com join auth.users.email
- `admin_list_alarm_audit_config()` — lista users monitorados + log_count
- `admin_toggle_alarm_audit(email, enabled, notes)` — toggle por email

**Cron cleanup:** `alarm-audit-cleanup-daily` 3:15 UTC, retention 7d via `cron_alarm_audit_cleanup()`.

**Seed:** `lhenrique.pda@gmail.com` enabled = true.

### 2. JS helper (`src/services/notifications/auditLog.js`)

- `isEnabled()` cache 5min via RPC
- `logAuditEvent(ev)` + `logAuditEventsBatch(events)` silent-fail
- Wire `scheduler.js rescheduleAll`:
  - `batch_start` (dosesCount, patientsCount)
  - per-dose `scheduled` (kind=critical_alarm OR local_notif)
  - per-dose `skipped` (reason=dnd_window)
  - `batch_end` (alarmsScheduled, dndSkipped, localNotifs, groupsCount)

### 3. Java helper (`AlarmAuditLogger.java`)

- Executor single-thread (não bloqueia AlarmScheduler hot path)
- `logScheduled/logCancelled/logFired/logBatch` POST `/rest/v1/alarm_audit_log`
- Credentials reused `dosy_sync_credentials` SharedPrefs
- `device_id = Build.MODEL + " (" + Build.MANUFACTURER + ")"`

**Wires:**
- `DoseSyncWorker.scheduleDoses()` — batch_start/end + per-dose scheduled (source `java_worker`)
- `DosyMessagingService.handleScheduleAlarms()` — batch source `java_fcm_received` + per-dose scheduled
- `DosyMessagingService.handleCancelAlarms()` — per-dose cancelled (reason=fcm_cancel_action)
- `AlarmReceiver.onReceive()` — per-dose fired_received (source `java_alarm_scheduler`)

### 4. Edge Functions (`_shared/auditLog.ts` novo)

- `getEnabledAuditUsers(supabase)` → `Set<user_id>` whitelist preload
- `logAuditBatch(supabase, rows[])` insert batch

**Wires:**
- `daily-alarm-sync` v2 — per-user gate + batch_start/end + per-dose fcm_sent/skipped (source `edge_daily_sync`)
- `dose-trigger-handler` v17 — per-dose fcm_sent/cancelled/skipped (source `edge_trigger_handler`) com `triggerType` (INSERT/UPDATE/DELETE) + `triggerReason` (insert_pending_future / status_pending_to_done / scheduled_at_changed / undo_to_pending / delete_pending) no metadata

## Egress audit

| Cenário | Rows/dia estimado lhenrique |
|---|---|
| JS rescheduleAll (3x/dia) | ~60 rows (14 doses × 3 + batch tags) |
| Worker periodic 6h (4x/dia) | ~60 rows |
| FCM received cron 5am + trigger ~10/dia | ~150 rows |
| AlarmReceiver fired (14/dia) | 14 rows |
| Edge daily-sync 5am | ~30 rows |
| Edge trigger-handler real-time ~10/dia | ~10 rows |
| **Total estimado** | **~324 rows/dia/device habilitado** |
| Outros users (não habilitados) | 0 rows |

7d retention × 324/dia = ~2300 rows total — trivial pra Postgres.

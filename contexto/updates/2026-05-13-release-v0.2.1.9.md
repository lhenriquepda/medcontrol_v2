## Motivação

User-reported 2026-05-13 manhã: 3 bugs simultâneos S25 Ultra Android 16:
1. Alarme 20h 2026-05-12 tocou com várias doses no horário mas header "Sem Paciente"
2. Push notification 5am 2026-05-13 pra Broncho-Vaxom Liam/Rael (agendado 8am)
3. Alarme 8am 2026-05-13 NÃO tocou — Broncho-Vaxom + 14 outras doses regulares 8am

## Causas raiz

**Bug 1** — `DoseSyncWorker.java` query `/rest/v1/doses` sem embed `patients(name)`. Hardcoded `patientName=""` quando Worker era fonte do scheduling. AlarmActivity render "Sem Paciente" fallback.

**Bug 2** — RPC `update_treatment_schedule` PL/pgSQL sem `AT TIME ZONE`. Interpretava `dose_times = "08:00"` como UTC. Doses criadas com `scheduledAt = 05:00-03:00` (5am BRT) ao invés de 8am.

**Bug 3** — Arquitetura 5 paths redundantes sem coordenação (cron 1min + cron 6h + Worker + JS + trigger). Race + over-egress + Doze blocking S25 Ultra → alarmes locais nunca agendados pra ids específicos.

## Implementação

### Fase 1 — Fixes imediatos

- `DoseSyncWorker.java`: query PostgREST embed `patients(name)` + extrai `patientName`. HORIZON 168h → 48h.
- `AlarmScheduler.java`: novo `cancelAlarm(ctx, id)` static + `removePersisted` helper.
- `DosyMessagingService.java`: novo handler action `cancel_alarms`.
- DB migration `fix_update_treatment_schedule_timezone` — RPC com param `p_timezone DEFAULT 'America/Sao_Paulo'`, `v_start_local := (startDate AT TIME ZONE p_timezone)::timestamp`, `v_first := (date_trunc('day', v_start_local) + make_interval(...)) AT TIME ZONE p_timezone`.
- DB migration `data_fix_doses_timezone_v0_2_1_9_retry` — regenera doses `status=pending scheduledAt > now()` via RPC fixada (idempotente).

### Fase 2 — Refactor cron 5am

- Nova Edge Function `daily-alarm-sync/index.ts` — cron `0 8 * * *` UTC (5am BRT), horizon 48h, FCM data retry exponencial 3 attempts, multi-TZ via `user_prefs.prefs.timezone`. Shared `_shared/userPrefs.ts`.
- Edge `dose-trigger-handler` v16 — horizon 6h → 48h, novo action `cancel_alarms` (DELETE + UPDATE pending→non-pending + scheduledAt-change).
- DB migration `cron_jobs_v0_2_1_9_daily_alarm_sync` — UNSCHEDULE `notify-doses-1min` + `schedule-alarms-fcm-6h`. SCHEDULE `daily-alarm-sync-5am`.

### Fase 3 — Bug #208 inline fix

- `useAppUpdate.js` — VERSION_CODE_TO_NAME +56 +57 entries (UpdateBanner versionName correto).
- Memory note `feedback_release_lifecycle.md` — checklist obrigatório bump map a cada release.

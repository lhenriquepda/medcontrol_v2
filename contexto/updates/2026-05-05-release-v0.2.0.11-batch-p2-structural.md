## Items fechados v0.2.0.11

### #144 — Custom JWT claim tier (Auth Hook)

**Migration `144_jwt_claim_tier_auth_hook`**:
- Schema `auth_hooks` criado
- Function `auth_hooks.add_tier_to_jwt(event jsonb) returns jsonb` chama `medcontrol.effective_tier(uid)` e injeta em `claims.app_metadata.tier`
- Grants supabase_auth_admin EXECUTE
- REVOKE de authenticated/anon/public

**Dashboard hook habilitado** (Auth → Hooks → Customize Access Token Claims) via Chrome MCP:
- Schema: `auth_hooks`
- Function: `add_tier_to_jwt`
- Status: ✅ ENABLED

**Frontend `getMyTier()`**:
- Antes: `supabase.auth.getUser()` round-trip + `supabase.rpc('my_tier')` round-trip = 2 calls servidor
- Agora: `supabase.auth.getSession()` (local-only) + lê `user.app_metadata.tier` (claim local) = 0 calls
- Fallback gracioso: legacy session sem hook → 1× rpc + force `refreshSession()` p/ próximas leituras virem do claim
- staleTime ampliado 30min → 1h (refresh natural quando JWT renova)

**`useGrantTier()` aprimorado**: pós-grant em si próprio, força `refreshSession()` + invalidate. Cross-user requer logout/login do impactado.

**Estimado: -90% calls /auth/v1/user + -100% rpc(my_tier)**.

### #145 — useRealtime watchdog scoped invalidate

**`src/hooks/useRealtime.js`**:
- `onStatusChange` reconnect path: `qc.invalidateQueries(...)` blanket → `qc.refetchQueries({type:'active'})` scoped

## Risco/regressão

- **#144 Auth Hook**: legacy sessions (pré-deploy) caem em fallback rpc 1× + refresh. Próximas sessions OK. Zero regressão funcional, só path lento extra na 1ª sessão pós-deploy.
- **#145 scoped refetch**: queries não-active não atualizam após reconnect — mas refetch natural ao montar resolve. UX: dados eventualmente fresh sem cost upfront.
- **#029/#030 splits**: Build verde. JSX layout idêntico (copiado char-by-char). Risk só se import path circular surpresa, mas Vite build verde = OK.
- **#034 virtualização**: pode quebrar se row height muito variável. Mitigação: ref + measureElement pra ajustar. listas curtas (<10) não scrollam (overflow visível).
- **#100 avatar**: legacy patients com `'👤'` salvo continuam funcionar (fallback `||` garante). Visual: opt-in via re-edit pro novo emoji.

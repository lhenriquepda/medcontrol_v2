## Bugs descobertos durante validação preview Vercel + fixes

### 🚨 #144 hook causou logout cascade em prod — ROLLBACK

**Sintoma:** login ok → carrega dados → desloga sozinho. teste-plus + lhenrique.pda admin ambos afetados.

**Root cause:** `getMyTier()` com `refreshSession()` automático em fallback path:
1. Hook ativo emit JWT com `app_metadata.tier` claim (testado SQL OK)
2. Sessão legacy pré-deploy em localStorage não tem claim → fallback path
3. Frontend chama `rpc('my_tier')` + dispara `refreshSession()` non-blocking
4. `refreshSession()` emit TOKEN_REFRESHED event
5. `useAuth.jsx:56` listener: `qc.clear()` em mudança session
6. Cache limpa → todas queries refetch → useMyTier re-roda → claim ainda undefined → fallback again → loop
7. Cascade: refresh storm → rate limit `/token` 429 → AuthContext signOut

**Fix imediato:**
- Hook **DISABLED** Dashboard via Chrome MCP (toggle off)
- Frontend `getMyTier()` revertido pra path simples (sem refreshSession)
- Migration permanente (function intacta no DB)
- Plan v0.2.0.12: re-implementar conservador (only read claim, no auto-refresh)

### 🚨 #148 extend_continuous_treatments rpc 2× por mount Dashboard

**Sintoma:** monitor V3 (intercept fetch) mostrou 2 calls simultaneous `+0ms` cada navegação Dashboard.

**Root cause:** `AnimatePresence mode="popLayout"` mantém old + new Dashboard durante exit anim ~600ms. Both useEffects fire = 2 rpc calls.

**Fix:** Module-scope debounce 60s via `window.__dosyExtendContinuousAt`. Skip se chamou nos últimos 60s. RPC idempotente, mas dobra cost por mount.

### 🚨 #150 useDoses refetchInterval idle storm (5 queryKeys × 5min)

**Sintoma:** monitor V3 5min IDLE detectou rajada 5 GET /doses simultâneos `+282s` (50KB) sem nenhuma interação user.

**Root cause:** `useDoses` `refetchInterval: 5 * 60_000` dispara em **TODAS active queryKeys cached** independentemente:
- Dashboard base (-30d/+60d) ~15KB
- Settings (próximas 48h) ~2KB
- DoseHistory (-7d) ~2.5KB
- Reports (30d+obs) ~15KB × 2

5 active queryKeys × 50KB × 12 cycles/h × 24h × 1000 users = **14GB/dia idle polling**.

**Fix:** `refetchInterval` 5min → 15min = -67% polling rate. Realtime postgres_changes + watchdog #079 cobrem reconnect/silent-fail. Mutation invalidate (#149 debounced) cobre user changes.

**Estimado pós-#150:** 4.7GB/dia idle (vs 14GB) em 1000 users hipotéticos.

### 🚨 #149 useDoses refetch storm pós-mutation (3+ active queryKeys)

**Sintoma:** 200s sessão real (mark tomada + skip + undo) gerou 12 GET /doses (99KB egress).

**Root cause:** Cada mutation `confirm/skip/undo` em `onSettled` chama `refetchDoses(qc)` = invalida ['doses'] → todas active queryKeys refetcham. Multi-query app (Dashboard -30d/+60d, Settings 48h, DoseHistory 7d, Reports 30d+obs) = 3-4 queryKeys ativas → 3-4 fetches por mutation. Multi-mutation rapid (4 clicks) = 12-16 fetches.

**Fix:** Debounce refetchDoses 2s via module-scope timer. Múltiplas mutations rapid (confirm→undo→skip) consolidam em 1 refetch único pós-2s idle. Optimistic update via `patchDoseInCache` já garante UI consistency, refetch é safety net.

**Estimado redução:**
- Burst 4 mutations: 12 → 3 fetches /doses (-75% storm)
- Single mutation: 3 → 3 (sem mudança)
- User típico: -60% to -75% egress mutation-induced

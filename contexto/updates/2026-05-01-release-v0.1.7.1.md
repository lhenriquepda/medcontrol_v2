## 🧠 Decisões importantes

- **Foreground service descartado** pelo user. Evita notif persistente "Dosy rodando" 24/7 que afetaria UX.
- **Force-stop como limitação documentada** — não bloqueia release. Cenário extremo (user deliberadamente mata app via Settings). Onboarding "permita background" pra mitigar futuramente.
- **Catalogada release v0.1.7.2 (#083)** — FCM-driven scheduling com 4 caminhos coordenados. Trigger DB ON INSERT + cron 6h FCM data + rescheduleAll quando app abre + WorkManager. Push tray inteligente (skip se alarme nativo agendado). Filosofia "se dose tem alarme, ok; senão, push como fallback".
- **Dual-app permanente** — Dosy Dev pra release branches (debug variant), Dosy oficial pra master/Play Store (release variant).

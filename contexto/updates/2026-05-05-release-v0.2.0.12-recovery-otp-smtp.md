### #144 Auth Hook tier claim re-ativado (P2 longo prazo)

**Backend permanente desde v0.2.0.11:**
- Migration `144_jwt_claim_tier_auth_hook` (schema `auth_hooks` + função `add_tier_to_jwt`)
- Hook injeta `app_metadata.tier` em JWT claims via `medcontrol.effective_tier(uid)`

**Bug crítico v0.2.0.11:**
- qc.clear() em onAuthStateChange disparava em TOKEN_REFRESHED
- Hook ativo: getMyTier fallback → refreshSession → TOKEN_REFRESHED → qc.clear → loop logout cascade
- Frontend rollback v0.2.0.11 (hook DESABILITADO Dashboard)

**Fix conservador v0.2.0.12 (commit 54e0d0a):**
- `useAuth.jsx onAuthStateChange`: qc.clear() SCOPED apenas em:
  - `SIGNED_OUT` (logout)
  - `SIGNED_IN` com `prevUserId !== u?.id` (admin switch user, raro)
- TOKEN_REFRESHED preserva cache (mesmo user, mesmo dado)
- INITIAL_SESSION + boot SIGNED_IN: prevUserId null → não clear

**Estado prod:** Hook DISABLED Dashboard até master merge v0.2.0.12. Pós-merge, re-ativar manual via Dashboard → Auth → Hooks → toggle "Customize Access Token (JWT) Claims hook". Ganho: -100% rpc('my_tier') round-trip.

### Tradução erros expandida (commit 9dfb0f5)

`src/services/supabase.js` `traduzirErro()` adicionou 7 regex matchers:
- invalid/expired/incorrect OTP token code → "Código inválido ou expirado..."
- token has expired → "Código expirado. Peça um novo código."
- otp/token not found → "Código não encontrado..."
- email rate limit → "Muitas solicitações. Aguarde 1 minuto..."
- over request rate limit → "Muitas tentativas. Aguarde alguns minutos."
- user not found → "Email não cadastrado."
- signup disabled → "Cadastros desativados. Contato suporte."

Antes: erro Supabase OTP caía em fallback genérico "Ocorreu um erro. Tente novamente." User não tinha pista.

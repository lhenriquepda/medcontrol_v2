## Items fechados v0.2.0.8

### Auditoria egress robusta — `contexto/egress-audit-2026-05-05/`

Análise linha-a-linha do consumo egress:
- Dia 04 May 2026: 9.64 GB (PostgREST 98.6%, Realtime 1.4%, Auth 0%)
- Cache hit ratio: 0% (nada cacheado)
- pg_stat_statements top: `SELECT 11cols FROM doses WHERE scheduledAt BETWEEN...` — 3.5M calls / 1.4 calls/s contínuo

**Causa raiz identificada:** invalidate cascade em events não-data-related. Múltiplos hooks (useAppResume, useRealtime watchdog/resume/postgres_changes/native lifecycle) disparando `qc.invalidateQueries()` sem coordenação → refetch storm em qualquer visibility/focus/resume change.

13 items adicionados ROADMAP/CHECKLIST (#134-#146): P0 cost (5 items, esforço ~4h, impacto -65% a -75%) + P0 security (1 item) + P1 cost (3 items) + P2 estrutural (4 items).

### Fixes P0 aplicados (esta release)

**#134 — `useAppResume`: short idle não invalida mais**
- `src/hooks/useAppResume.js`: REMOVIDO `qc.invalidateQueries()` em short idle (<5min)
- Long idle (>=5min): `refetchQueries({ active })` sem invalidate redundante
- Realtime postgres_changes + refetchInterval 5min cobrem updates necessários
- Estimado: **-30% a -45% egress**

**#135 — `useRealtime` resume nativo: invalidate ALL removido**
- `src/hooks/useRealtime.js`: removido loop `qc.invalidateQueries` em CapacitorApp.resume
- Resubscribe + Realtime postgres_changes events trazem updates pós-resume
- Estimado: **-5% a -10% egress**

**#136 — `useRealtime` postgres_changes: debounce 1s**
- `src/hooks/useRealtime.js`: wrap `qc.invalidateQueries` em Map<key, timeout> debounce 1000ms
- Cleanup pending timers no useEffect return
- Cron `extend_continuous_treatments` que insere 100s doses em batch agora dispara 1 invalidate vs 100
- Estimado: **-15% a -25% egress (especialmente dias de cron)**

**Total estimado P0: -50% a -75% egress.**

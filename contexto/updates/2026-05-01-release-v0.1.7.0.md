## 🚨 Achado live nav: BUG-016 healthcare-critical

Validação device físico (teste03 em Android com APK debug v0.1.7.0):
- 18:37 ✅ alarme + push OK (app ativo)
- 18:39 ✅ multi-dose ok (app ativo)
- **18:55 ❌** após ~16min idle: nada disparou. Sem alarme, sem push, sem notif tray. Dose ficou "Atrasada" só visível ao user reabrir app.
- 18:58 ✅ alarme-off teste: push chegou (mas user tinha acabado de mexer no app)

**User feedback CRITICAL:**
> "essa questão do alarme não tocar e push não ir quando app fica em idle muito tempo é critico... idoso não costuma fechar aplicativo NENHUM... o tempo de idle deve ser ilimitado e ainda assim funcionar alarme e push"

**Diagnóstico:**
- Hoje há **1 caminho único** de notificação: app foreground / recente-ativo → realtime websocket recebe `postgres_changes` → app agenda alarme nativo local. Se algum elo falha (websocket morre, app killed em background), tudo cai.
- BUG-016 vira release inteira `v0.1.7.1` com **defense-in-depth 3 caminhos independentes**:
  - **#079** (caminho 1): heartbeat realtime + auto-reconnect
  - **#080** (caminho 2): `notify-doses` Edge cron robustness (retry, cleanup tokens, observability)
  - **#081** (caminho 3): alarmes nativos horizonte 24-72h via WorkManager — independe de qualquer caminho web

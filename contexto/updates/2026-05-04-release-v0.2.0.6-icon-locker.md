## Items fechados

### #010 — `ic_stat_dosy` notification icon (P1 BUG-005)
Commit `cbfc813`.

**Problema:** `ic_stat_dosy.png` em drawable/ era 96x96 colorida (gradient peach). Android API 21+ só respeita alpha em smallIcon — color renderiza bloco sólido em status tray. Os 3 paths Java nativos (AlarmReceiver, AlarmActivity, AlarmService) usavam `R.mipmap.ic_launcher` full color = mesmo bug.

**Fix:**
- Criado `drawable/ic_stat_dosy.xml` vector 24dp (capsule pill silhouette branca, fillColor #FFFFFFFF — single source escala todas densidades).
- Removido `drawable/ic_stat_dosy.png` stale.
- 3 arquivos Java: `setSmallIcon(R.drawable.ic_stat_dosy)` + `setColor(0xFFFF6B5B)` accent peach (Android tinta o silhouette).
- LocalNotifications via capacitor.config.ts `smallIcon: 'ic_stat_dosy'` agora resolve corretamente.

**Validação device S25 Ultra (2026-05-04):** ✅ notificação dispara, status tray mostra silhueta correta (não bloco sólido).

### #017 — LockScreen UI + biometria (P1 Plan FASE 11.3)
Commit `869ab34`.

**Implementado:**
- `src/components/LockScreen.jsx` (novo): overlay full-screen com auto-trigger biometric on mount + retry button + "Sair da conta" escape hatch.
- `src/App.jsx`: import useAppLock + render LockScreen quando locked=true (precede loading/routing).
- `src/hooks/useAppLock.js`: refactor — `timeoutMin` agora useState (era `readTimeout()` inline → Dropdown não atualizava). `disable()` retorna bool (toast). `setTimeoutMin` sincroniza state + localStorage.
- `src/pages/Settings.jsx`: nova seção "Privacidade e segurança" (native-only) com Toggle "Bloqueio do app" + Dropdown timeout 1/5/15/30/60min default 5.

**Comportamento:**
- Toggle ON → biometria solicitada (anti-tamper) → enable → app bloqueia próximo cold start + após N min em background.
- Toggle OFF → biometria re-solicitada → disable. Cancelar mantém ativo (proteção).
- BiometricAuth `allowDeviceCredential: true` → fallback PIN/senha celular se biometria não enrolada (sem PIN app-internal).
- Web: hook no-op, seção Settings escondida.

**Validação device S25 Ultra (2026-05-04):** ✅ 6 testes passaram —
1. Toggle ON exige biometria + toast "Bloqueio ativado" + dropdown timeout aparece
2. Mudar timeout pra 1 min funciona
3. Cold start (force close + reabrir) → tela de bloqueio + prompt biometria → unlock
4. Cancelar prompt (back button — Samsung S25 Ultra não mostra "Cancelar" explícito quando allowDeviceCredential: true) mantém locked + botões Desbloquear/Sair-da-conta disponíveis
5. Auto-lock após 1 min em background
6. Toggle OFF exige biometria (anti-tamper)

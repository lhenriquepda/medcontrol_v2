## Items fechados

### Dosy Dev FLAG_SECURE off (debug variant `.dev` package)

Commits `41f7200` + `9c69202`.

**Problema:** plugin `@capacitor-community/privacy-screen` 8.0.0 aplica FLAG_SECURE automaticamente no `load()` nativo (constructor da classe `PrivacyScreen` chama `addFlags()` se config.enabled — default true). Hook `usePrivacyScreen` chama `enable()` em telas sensíveis mas FLAG já estava ativo desde boot — irrelevante o hook saber ou não.

**Fix duplo:**
1. `src/hooks/usePrivacyScreen.js`: detecta package via `Capacitor.App.getInfo()` async — se `.dev` → return early (não chama enable() em desmonts/cleanup).
2. `src/main.jsx`: bloco early no boot detecta package — se `.dev` → import + `PrivacyScreen.disable()` explícito (cancela o flag aplicado pelo plugin load nativo).

**Comportamento por variant:**
- Dosy Dev (`com.dosyapp.dosy.dev`, debug): FLAG_SECURE OFF — screenshots + screen recording funcionam.
- Dosy oficial (`com.dosyapp.dosy`, release): FLAG_SECURE ON — bloqueado normal (info médica protegida).
- Web: hook já era no-op.

### StatusBar sincronizado com tema (light/dark)

Commit `10859da`.

**Problema:** `main.jsx` forçava `Style.Dark` (light text) + `setBackgroundColor('#0d1535')` (azul escuro legacy pré-redesign peach v0.2.0.0). Em modo claro do app: ícones brancos sobre bg azul não combinavam + invisíveis em algumas viewports.

**Fix:**
- `main.jsx`: removido override hardcoded. Mantido só `setOverlaysWebView(false)`.
- `src/hooks/useTheme.jsx`: useEffect sincroniza StatusBar com `theme`:
  - `light` → `Style.Light` (dark icons) + bg `#FFF4EC` (peach)
  - `dark` → `Style.Dark` (light icons) + bg `#1A0F0A` (peach-dark)
- Re-aplica em qualquer toggle.

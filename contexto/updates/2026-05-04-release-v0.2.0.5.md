## Items fechados

### #126 [P0 SECURITY] Pre-commit secret scanning + root cause analysis

**Causa raiz**: husky pre-commit só rodava `npx lint-staged` (eslint). Zero scan de secrets. Padrões dev:
- `tools/*.cjs` hardcoding connection strings em vez de `process.env`
- `Contexto.md` documentando env vars com VALORES REAIS em vez de placeholders

**Fixes**:

1. `gitleaks` instalado (scoop 8.30.1)
2. `.gitleaks.toml` config:
   - Default rules + custom regexes (postgres conn string, VAPID, supabase service role JWT)
   - Allowlist: placeholders, `.env*`, `.gitleaks.toml`, build artifacts, `google-services.json` (convenção mobile)
3. `.husky/pre-commit` reescrito:
   - `gitleaks protect --staged` PRIMEIRO (bloqueia secrets)
   - `lint-staged` depois
   - Mensagem clara em caso de bloqueio
4. `.github/workflows/gitleaks.yml`:
   - CI segunda camada (não-bypassable com `--no-verify`)
   - Roda em push de qualquer branch + PRs pra master
5. Full filesystem scan validado: 27 → 0 leaks após allowlist tuning

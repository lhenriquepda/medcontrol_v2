## Motivação

v0.2.2.2 (#212) aplicou signature guard em App.jsx mas storm 60s continuou. Auditoria via logcat Dosy-Dev (~6min monitoramento) capturou root cause real.

## Root cause (logcat evidência)

```
15:33:29 rescheduleAll START dosesCount=30 patientsCount=3
15:33:29 cancelAll
15:33:29 scheduled id=386571675 count=2
15:33:30 scheduled id=1912498711 count=7

15:34:31 rescheduleAll START dosesCount=30 (+62s — IDÊNTICO!)
... loop infinito
```

**Conteúdo idêntico entre cycles** — mesmos IDs, mesmas timestamps `at=...`. Pattern característico de re-mount cíclico.

### Cadeia identificada

`src/pages/Dashboard.jsx:99`:
```js
const t = setInterval(() => setTick(Math.floor(Date.now() / 60000)), 60000)
```

Plus `Dashboard.jsx:222-224`:
```js
const { scheduleDoses } = usePushNotifications()
useEffect(() => {
  scheduleDoses(todayDoses, { patients })
}, [todayDoses, patients, scheduleDoses])
```

**Mecanismo storm 60s:**
1. `setInterval 60s` → `setTick(novo número)`
2. `tick` muda → `todayDoses = useMemo(... [doses, tick])` recalcula → new array ref
3. `useEffect([todayDoses, ...])` re-fires → `scheduleDoses(todayDoses, {patients})`
4. `rescheduleAll` → `cancelAll` + reagenda 9 alarmes (mesmo conteúdo!)
5. 60s loop reinicia

### Por que signature guard App.jsx não pegou?

App.jsx signature guard funciona no App.jsx useEffect próprio. Dashboard.jsx tem seu próprio useEffect chamando `scheduleDoses` diretamente — bypassa App.jsx completamente. 2 callers competindo.

## Implementação

Fix mínimo (1 useEffect + 1 import removidos):

```diff
- import { usePushNotifications } from '../hooks/usePushNotifications'
-
- // Schedule push notifications for upcoming doses + daily summary.
- // SEMPRE roda (mesmo sem doses hoje) — daily summary é independente e precisa
- // ser agendado mesmo quando não há doses programadas.
- const { scheduleDoses } = usePushNotifications()
- useEffect(() => {
-   scheduleDoses(todayDoses, { patients })
- }, [todayDoses, patients, scheduleDoses])
+ // v0.2.2.3 (#213) — REMOVIDO scheduleDoses caller. App.jsx top-level useEffect
+ // (com signature guard v0.2.2.2) já agenda full window 48h. Dashboard caller
+ // era vestígio pré-#198 + executava a cada 60s via setInterval setTick:99
+ // que flipava todayDoses ref → useEffect re-fire → cancelAll + reagenda mesmo
+ // conteúdo idêntico → storm 1440 reschedules/dia. Audit logcat confirmou.
+ // Daily summary é agendado dentro do mesmo rescheduleAll de App.jsx, sem perda.
```

App.jsx top-level cobre full 48h window. Daily summary agendado dentro do mesmo rescheduleAll. dose-trigger-handler Edge cobre changes real-time.

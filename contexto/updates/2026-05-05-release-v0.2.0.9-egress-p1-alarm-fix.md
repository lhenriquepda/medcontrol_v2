### Bug fix: filter "Tudo" → "10 dias"

User reportou Dashboard "Tudo" não carrega. Root cause: `rangeNow('all')` retorna `{from:null, to:null}`. Em #137 Dashboard client-side filter:
```js
const fromMs = new Date(rangeFrom).getTime() // new Date(null).getTime() = 0
let r = allDoses.filter(d => {
  const t = new Date(d.scheduledAt).getTime()
  return t >= fromMs && t <= toMs // t >= 0 && t <= 0 → false
})
```

Comparison NaN/0 rejeita todas doses → array vazio.

Fix:
- `src/components/FilterBar.jsx`: removido entry 'Tudo'. Substituído por '10 dias' (key '10d').
- `src/utils/dateUtils.js`: rangeNow('10d') retorna janela -5d/+10d (proporção -N/2/+N).
- `src/pages/Dashboard.jsx:68,75`: setFilters URL flow setta range:'7d' (era 'all').
- FilterBar status click: range fallback '7d' (era 'all').

**User pediu '10 dias' especificamente** depois de inicialmente eu propor remover "Tudo" totalmente.

### Bug fix: useDoses queryKey withObservation

Fix: `[filter.from, filter.to, filter.patientId, filter.status, filter.type, filter.withObservation]`.

Cache válido agora por tuple completa.

## Bugs novos descobertos

### BUG-017 (carry-over → #084) — Service role JWT vazado em commit

**Severidade:** P0 security · **Quando:** 2026-05-02 22:23 UTC

Service role JWT foi commitado por engano em `supabase/migrations/20260502091000_dose_trigger_webhook.sql` (commit original `85d5e61`). GitGuardian + GitHub Security detectaram em ~6min.

**Mitigação imediata na branch:**
- Histórico do branch reescrito via `git-filter-repo`
- Force push pra remover do branch ativo
- Migration recriada com workaround: trigger function chama Edge que está deployada `--no-verify-jwt` (validação de origem dentro da Edge)

**Pendente (próxima sessão / hotfix v0.1.7.3):**
- Rotacionar JWT secret Supabase Dashboard → Settings → API → Roll JWT Secret
- Atualizar `VITE_SUPABASE_ANON_KEY` em Vercel + `.env.local` + rebuild apps
- Auditar logs Auth/REST janela 22:23-22:29 UTC pra detectar uso suspeito
- Tag `pre-secret-purge-backup` empurrada origin pra referência se precisar rollback

**Risco residual:** chave permanece em GitHub commit cache + indexers externos (Google cache, Wayback) por tempo indeterminado. Service_role bypassa RLS = exposição teórica de TODOS dados saúde de TODOS users (LGPD categoria especial). Rotação obrigatória antes próxima exposição pública.

## Bug fix collateral

**`firebase-messaging` faltando em compile classpath:**

`@capacitor/push-notifications` declara `firebase-messaging` como `implementation` (encapsulado). DosyMessagingService.java em `:app` não enxergava `RemoteMessage`. Release variant build falhou com `cannot find symbol class RemoteMessage`. Fix: declarar `implementation "com.google.firebase:firebase-messaging:25.0.1"` direto em `android/app/build.gradle` (mesma versão do plugin).
